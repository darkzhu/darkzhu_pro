# 个人博客项目技术选型建议

## 1. 目标

选择一套适合个人维护、开发体验较好、部署方便、支持 Markdown 内容管理的技术方案。

## 2. 推荐方案

推荐优先方案：

- 框架：Next.js
- 语言：TypeScript
- 样式：Tailwind CSS
- 内容来源：本地 Markdown 或 MDX
- 部署平台：Vercel
- 评论系统：Giscus
- 数据统计：Umami 或 Google Analytics

## 3. 方案说明

### 3.1 Next.js

优点：

- 生态成熟
- 适合博客、内容站点、作品集网站
- 支持静态生成，SEO 友好
- 部署到 Vercel 成本低，流程顺畅
- 后续扩展 API、订阅、搜索等功能也方便

适用场景：

- 想做一个长期维护的个人博客
- 后续可能同时做作品集和个人主页

### 3.2 TypeScript

优点：

- 提升可维护性
- 对后续扩展更稳
- 对组件、数据结构、文章元信息约束更清晰

### 3.3 Tailwind CSS

优点：

- 开发效率高
- 容易快速搭建页面
- 后续统一样式方便

注意：

- 如果你更重视强设计表达，也可以在 Tailwind 基础上配合自定义 CSS 变量和组件样式

### 3.4 Markdown / MDX

推荐原因：

- 写作简单
- 内容与代码分离清晰
- 适合个人博客
- Git 管理方便

区别：

- Markdown：更简单，适合纯文章
- MDX：支持在文章中嵌入 React 组件，扩展性更强

建议：

- 如果是第一次做博客，用 Markdown 即可
- 如果你希望以后做更丰富的文章排版，选 MDX

### 3.5 Vercel

优点：

- 与 Next.js 集成最好
- 自动部署方便
- 适合个人项目快速上线

### 3.6 Giscus

优点：

- 基于 GitHub Discussions
- 无需自建评论后端
- 风格比较适合技术博客

## 4. 备选方案对比

### 方案 A：Next.js + Markdown

适合：

- 开发者博客
- 兼顾展示与扩展

结论：

- 综合最推荐

### 方案 B：Astro + Markdown

适合：

- 更偏内容站
- 更看重性能和静态输出

优点：

- 生成静态页面很强
- 默认输出轻量

结论：

- 如果你极度重视静态博客体验，也值得选

### 方案 C：Hugo

适合：

- 不想花太多时间在前端工程上
- 更偏传统博客生成器

结论：

- 上手快，但自定义复杂交互时没有 Next.js 灵活

## 5. 最终建议

如果你要做的是一个能长期扩展的个人博客项目，建议选：

`Next.js + TypeScript + Tailwind CSS + MDX/Markdown + Vercel + Giscus`

这是比较稳、开发体验也好的组合。

## 6. 初始依赖建议

- `next`
- `react`
- `react-dom`
- `typescript`
- `tailwindcss`
- `gray-matter`
- `remark`
- `remark-html` 或 `next-mdx-remote`
- `clsx`
- `dayjs`

## 7. 后续增强依赖

- `shiki` 或 `prismjs`：代码高亮
- `fuse.js`：本地搜索
- `next-sitemap`：站点地图
- `@giscus/react`：评论
