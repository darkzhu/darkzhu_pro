# 为什么先做一个最小版本

做个人博客时，最容易卡住的不是代码，而是想一次性把所有功能都做完。

更稳的做法是先建立一个能写、能发、能改的最小版本，再逐步加功能。

## 这个初始框架已经做了什么

- 配好了基础页面
- 接好了 Markdown 文章读取
- 支持文章列表和详情页
- 保留了继续扩展评论、搜索、标签页的空间

## 接下来你可以做什么

1. 修改站点名称和简介
2. 改首页文案和关于页内容
3. 新增自己的第一篇正式文章

```ts
export function startWriting() {
  return "Keep shipping.";
}
```
